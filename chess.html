Move Validation

Logic

The engine of the game. Each piece type has its own movement predicate. Sliding pieces (Rook, Bishop, Queen) share a path-clearance helper.

Pawn special
2-sq start, diag capture
Knight
|dr|+|dc| = 3, ≠ straight
Sliding
IsClearPath() helper
Pawn: forward 1 (or 2 from start row), captures diagonally only
Knight: (|dr|==2 && |dc|==1) || (|dr|==1 && |dc|==2)
Bishop: |dr| == |dc|, clear diagonal path
Rook: same row or same col, clear straight path
Queen: Bishop OR Rook rules
King: |dr|≤1 && |dc|≤1, never friendly-occupied
bool IsClearPath(int fr, int fc, int tr, int tc) { int dr = (tr>fr)?1:(tr<fr)?-1:0; int dc = (tc>fc)?1:(tc<fc)?-1:0; // walk squares between src and dst for (int r=fr+dr,c=fc+dc; r!=tr||c!=tc; r+=dr,c+=dc) if (board[r][c] != EMPTY) return false; return true; }

Legal Move Highlights

UX Render

When a piece is selected, iterate every cell and highlight those that pass IsLegalMove(). Dots for empty squares, rings for capturable enemies.

Loop all 64 squares, call IsLegalMove(sel, target)
Empty target → small filled circle (radius ≈ 18% cell)
Enemy target → hollow ring (radius ≈ 38–46% cell) in red
King in check → full red tint on king's cell
// empty square hint DrawCircle(cx, cy, CELL_SIZE*0.18f, COLOR_MOVE_DOT); // capture ring DrawRing((Vector2){cx,cy}, CELL_SIZE*0.38f, CELL_SIZE*0.46f, 0, 360, 20, COLOR_CAPTURE);

Check & Checkmate

Logic

A move is only truly legal if it doesn't leave your own king in check. Simulate the move, test, then undo. Build on that to detect checkmate and stalemate.

IsInCheck
any enemy attacks King?
IsLegalMove
valid + !selfCheck
Checkmate
inCheck && !anyLegal
FindKing(color) — scan board for King piece
IsInCheck(color) — any enemy IsValidMove → King?
Simulate move, run IsInCheck, restore board (3 lines)
HasAnyLegalMove(color) — brute-force all 64×64 pairs
Checkmate = inCheck + no legal moves; Stalemate = opposite
